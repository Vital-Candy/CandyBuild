# CandyBuild

A lightweight, Gradle-free Android build system that runs entirely on-device, inside [Termux](https://termux.dev). No Android Studio, no Gradle daemon, no desktop required — write code on your phone, build an APK on your phone.

## Features

- **Kotlin, Java, or both** — mixed-language projects compile out of the box.
- **Jetpack Compose** — optional, resolved via a built-in Maven Central client.
- **Native C++ / JNI** — for games and performance-critical code, compiled against a real Android target (not Termux's own userland).
- **Maven dependencies** — declare `dependency = "group:artifact:version"` in `Candy.toml`, CandyBuild resolves, downloads, and merges it (classes + resources), transitively.
- **Full pipeline** — resources → R.java → Kotlin/Java → native → DEX → package → zipalign → sign, all from one command.

## Install

```bash
pkg install -y git && git clone https://github.com/Vital-Candy/CandyBuild ~/.candybuild && bash ~/.candybuild/install.sh
```

This installs OpenJDK, Kotlin, `aapt2`/`d8`/`apksigner`/`zipalign`, `clang` + `ndk-sysroot` for native code, and links the `candybuild` command. Run `candybuild doctor` afterward to confirm everything is in place — you'll still need an Android SDK `platforms/android.jar` (see [docs/SDK.md](docs/SDK.md)).

## Quick start

```bash
candybuild new MyApp      # scaffolds a project — asks for language, UI, native code
candybuild build MyApp    # produces MyApp/out/app-debug.apk
candybuild doctor         # checks your toolchain
```

## `Candy.toml`

Every project has one, generated by `candybuild new`:

```toml
package="com.example.myapp"
app_name="MyApp"
min_sdk=26
target_sdk=34
version_code=1
version_name="1.0"
language="kotlin"     # or "java"
ui="views"            # or "compose"

dependency="androidx.compose.material:material:1.6.7"
```

## Project layout

```
MyApp/
├── Candy.toml
├── AndroidManifest.xml
├── res/                  # layouts, values, drawables
├── src/                  # .kt / .java, mixed freely
├── cpp/                  # optional — native-lib.cpp etc.
├── assets/
├── libs/                 # prebuilt .so, if any
└── out/                  # built APKs land here
```

## Commands

| Command | Description |
|---|---|
| `candybuild new [name]` | Create a new project |
| `candybuild build [path]` | Build an APK (`--release` for a signed release build, `--no-clean` to skip wiping `build/`) |
| `candybuild setup <path>` | (Re)generate the standard folder layout |
| `candybuild clean [path]` | Remove `build/` and `out/` |
| `candybuild doctor` | Check the toolchain and SDK |

## Philosophy

No Gradle, no Android Studio, no heavy dependency graph. Every step is a plain, readable bash script calling standard Android command-line tools directly. If a build fails, you can read the exact command that failed — there's no black box in between.
